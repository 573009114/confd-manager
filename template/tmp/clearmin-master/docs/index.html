1. Quick start

To start using Clearmin template in a new project you can use this minimal template :

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1">
    <link rel="stylesheet" type="text/css" href="assets/css/bootstrap-clearmin.min.css">
    <link rel="stylesheet" type="text/css" href="assets/css/roboto.css">
    <link rel="stylesheet" type="text/css" href="assets/css/material-design.css">
    <link rel="stylesheet" type="text/css" href="assets/css/small-n-flat.css">
    <link rel="stylesheet" type="text/css" href="assets/css/font-awesome.min.css">
    <title>Clearmin Page</title>
  </head>
  <body class="cm-no-transition cm-1-navbar">
    <div id="cm-menu">
      <nav class="cm-navbar cm-navbar-primary">
        <div class="cm-flex"><div class="cm-logo"></div></div>
        <div class="btn btn-primary md-menu-white" data-toggle="cm-menu"></div>
      </nav>
      <div id="cm-menu-content">
        <div id="cm-menu-items-wrapper">
          <div id="cm-menu-scroller">
            <ul class="cm-menu-items">
              <li class="active"><a href="#" class="sf-house">This page is active</a></li>
            </ul>
          </div>
        </div>
      </div>
    </div>
    <header id="cm-header">
      <nav class="cm-navbar cm-navbar-primary">
        <div class="btn btn-primary md-menu-white hidden-md hidden-lg" data-toggle="cm-menu"></div>
        <div class="cm-flex"><h1>Put your title here</h1></div>
      </nav>
    </header>
    <div id="global">
      <div class="container-fluid">
        <div class="panel panel-default">
          <div class="panel-body">
            <h2 style="margin:0">Hello World !</h2>
          </div>
        </div>
      </div>
      <footer class="cm-footer"><span class="pull-right">&copy; ACME Inc.</span></footer>
    </div>
    <script src="assets/js/lib/jquery-2.1.3.min.js"></script>
    <script src="assets/js/jquery.mousewheel.min.js"></script>
    <script src="assets/js/jquery.cookie.min.js"></script>
    <script src="assets/js/fastclick.min.js"></script>
    <script src="assets/js/bootstrap.min.js"></script>
    <script src="assets/js/clearmin.min.js"></script>
  </body>
</html>
Note that every clearmin-specific classes and ids are cm- prefixed.

2. General structure

2.1. CSS and JS files

CSS files (<head>) :

  • assets/css/roboto.css (main font)
  • assets/css/bootstrap-clearmin.css (bootstrap theme)
  • assets/css/material-design.css (material design svg icons classes)
  • assets/css/small-n-flat.css (small-n-flat svg icons classes)
  • assets/css/font-awesome.min.css (iconic font classes)

Javascript files (just before </body>) :

  • assets/js/lib/jquery-2.1.3.min.js (required by bootstrap)
  • assets/js/lib/jquery.mousewheel.min.js (catch some scrolling events)
  • assets/js/lib/jquery.cookie.min.js (cookie manipulation plugin)
  • assets/js/lib/fastclick.min.js (enhance mobile device click events)
  • assets/js/bootstrap.min.js (bootstrap js widgets)
  • assets/js/clearmin.js (required by clearmin)

2.2. Body classes

  • cm-no-transition : must be present to prevent certain browser to start transition on page load
  • cm-1-navbar : when one navbar is present on your page
  • cm-2-navbar : when two navbars are present on your page
  • cm-3-navbar : when three navbars are present on your page
  • cm-menu-toggled : if you want the menu to be toggled on page load (see next section)

2.3. Preserving left menu state between pages

When the user choose to use the condensed version of the side menu, we want it to stay in this state when the user click on another page. To achieve this, a cookie is automatically set to reflect the menu state.

You got two different way to restore this state :

  • Server-side (recommended) : when your server receive a cookie named "cm-menu-toggled" with value "true", just add .cm-menu-toggled class to the body tag.
  • Client-side : Nothing to do, .cm-menu-toggled is automaticaly added to the body when needed but it can cause a blinking reflow on certain browsers (e.g. Google Chrome)

2.4. Content containers

  • Dont forget the <div id="global"> to contain the page (e.g. minimal template)
  • Use one or many <div class="container-fluid"> to contain your content

3. Navbars (header)

Here is the minimal code to build a Clearmin navbar (those must be placed inside <header id="cm-header">) :

<nav class="cm-navbar cm-navbar-default">
<div class="cm-flex"></div>
</nav>
  • .cm-flex will take all available space, other direct child elements will be square shaped (unless width is specified)
  • You can replace .cm-navbar-default class by one of our predifined color. (e.g. .cm-navbar-turquoise)

You can add as many buttons as you want, before and after .cm-flex.
You have to specify pull-right class when an item is on right
Several types of widgets can be inserted in it :

  • Breadcrumbs (default and primary navs only)
  • Tabs (default nav only)
  • Search Bar
  • Popovers
  • Dropdowns...

Check this demo page and pick what you need !

Add .cm-navbar-slideup class to the <nav> to hide it when user scroll down.

4. Left menu

4.1. Structure

Here is a basic example : 

<div id="cm-menu">
  <nav class="cm-navbar cm-navbar-primary">
    <div class="cm-flex"><div class="cm-logo"></div></div>
    <div class="btn btn-primary md-menu-white" data-toggle="cm-menu"></div>
  </nav>
  <div id="cm-menu-content">
    <div id="cm-menu-items-wrapper">
      <div id="cm-menu-scroller">
        <ul class="cm-menu-items">
          <li><a href="index.html" class="sf-house">Home</a></li>
        </ul>
      </div>
    </div>
  </div>
</div>

If you want to add some custom html content in that menu, you should include it into the <ul class="cm-menu-items"> to prevent eventual scroll bugs when menu content exceed page height.

4.2. Logo

You can change the brand logo by editing the last lines of bootstrap-clearmin.css / bootstrap-clearmin.min.css : 

#cm-menu .cm-logo {
  background: url(../img/your-logo-here.ext) no-repeat 15px center;
}

4.3. Submenu items

Only one level of submenu is currently supported, here is the code to add one :

<li class="cm-submenu">
  <a class="an-icon-class">My Submenu <span class="caret"></span></a>
  <ul>
    <li><a href="link1.html">Link 1</a></li>
    <li><a href="link2.html">Link 2</a></li>
    <li><a href="link3.html">Link 3</a></li>
  </ul>
</li>
To open a submenu on page load add the class .pre-open (<li class="cm-submenu pre-open">)

5. SVG icons

Clearmin provide a bunch of 24x24px svg icons in addition to the famous Font Awesome icons.
To use them as a background image, you can use classes present into assets/css/small-n-flat.css and assets/css/material-design.css.
To lighten those files, you may delete the classes you don't use.

6. Less Variables

Some variables in clearmin-global.less can be usefull to customize css file :

@cm-translen-global:150ms; // Transition duration for layout based animations
@cm-translen-widget:150ms; // Transition duration for modal, popovers, dropdown...
@cm-translen-button:75ms; // Transition duration for .btn items
@cm-menu-width:262px; // Left menu width
@cm-border-global:#ccc; // Default border color for several items
@cm-underliner-height:3px; // Height of underline in left menu and nav tabs
It's a good idea to customize @brand-primary in variables.less to reflect dominant color in an harmonious way.

7. Javascript Options

Some init options may be changed in clearmin.js / clearmin.min.js.

Here are the default options : 

$(function(){CM.init({
    'navbarHeight' : 50, /* should be changed reflecting @navbar-height less variable */
    'menuSwiper' : true, /* enable swipe gesture (toggle menu) */
    'menuSwiperIOS' : true, /* enable swipe gesture for ios devices */
    'restoreMenuState' : true /* activate client-side menu state restoration */
})});
